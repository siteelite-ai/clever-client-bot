

# План: передача контекста диалога в микро-LLM классификатор

## Проблема

Классификатор `classifyProductName` получает только текущее сообщение пользователя (`userMessage`). Когда бот задал уточняющий вопрос ("Какой тип фонаря?") и пользователь ответил "кемпинговый", классификатор не знает, что исходный запрос был "самый дорогой фонарь". Результат: `price_intent=none`, запрос идёт в обычный pipeline, ценовой интент теряется.

## Решение

Передавать в классификатор последние 2-4 сообщения из истории диалога вместо одного сообщения. Классификатор увидит:
- assistant: "В категории фонари 55 товаров. Какой тип?"
- user: "кемпинговый"

И правильно определит: `price_intent="most_expensive"`, `product_category="кемпинговый фонарь"`.

## Изменения в `supabase/functions/chat-consultant/index.ts`

### 1. Изменить сигнатуру `classifyProductName` (строка 514)

Принимать дополнительный параметр — последние сообщения из истории:

```typescript
async function classifyProductName(
  message: string, 
  recentHistory?: Array<{role: string, content: string}>
): Promise<ClassificationResult | null>
```

### 2. Обновить промпт классификатора (строки 536-559)

Добавить инструкцию: "Тебе может быть предоставлена недавняя история диалога. Если текущее сообщение — это ответ на уточняющий вопрос бота, учитывай ИСХОДНЫЙ запрос из истории для определения price_intent и product_category."

Формировать массив messages с историей:
```
[system prompt, ...recentHistory, {role: 'user', content: message}]
```

### 3. Передать историю при вызове (строка 2124)

Взять последние 4 сообщения из `historyForContext` (уже доступен в этой области видимости) и передать в `classifyProductName(userMessage, recentHistory)`.

### 4. Сохранять price_intent из clarify-контекста

Когда `priceIntentClarify` был установлен в предыдущем ходе — он теряется между запросами (каждый запрос — отдельный вызов функции). Поэтому именно история + классификатор должны восстанавливать контекст.

## Ожидаемый результат

| Диалог | Сейчас | После |
|---|---|---|
| Бот: "Какой тип фонаря?" → User: "кемпинговый" | price_intent=none, обычный pipeline | price_intent=most_expensive, category="кемпинговый фонарь", handlePriceIntent |
| Бот: "Уточните подкатегорию" → User: "налобный" | price_intent=none | price_intent сохраняется из контекста |

## Риски
- **Минимальный**: +50-100 токенов на классификацию (4 сообщения контекста), задержка ~0.1с
- **Нулевой**: первое сообщение без истории работает как раньше
