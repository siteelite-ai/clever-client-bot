

# План: Усиление промпта микро-LLM — «текущее сообщение всегда главнее»

## Что меняем

Одно изменение в файле `supabase/functions/chat-consultant/index.ts`, строка 580.

Добавляем **первой строкой** системного промпта классификатора жёсткое правило приоритета текущего сообщения:

```
ГЛАВНОЕ ПРАВИЛО: Определяй intent ТОЛЬКО по текущему сообщению пользователя.
История диалога — справочный контекст для коротких уточнений, НЕ для определения интента.
Если текущее сообщение содержит любые слова-товары (розетка, кабель, автомат, щит, лампа, выключатель, провод, удлинитель, счётчик, реле, контактор и т.д.) — intent ВСЕГДА "catalog", даже если все предыдущие сообщения были про оплату, доставку или прайс.
```

Это правило ставится ДО существующего текста промпта (до строки «Ты классификатор сообщений...»), чтобы LLM увидел его первым и придал ему максимальный вес.

## Объём

- 1 файл, ~5 строк добавления в начало промпта
- 1 деплой edge function

## Ожидаемый результат

Сценарий «как оплатить?» → «условия доставки» → «нужна черная двухместная розетка» — третье сообщение получает `intent=catalog`, несмотря на info-историю.
