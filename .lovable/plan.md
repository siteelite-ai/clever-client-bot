

# Диагноз: почему «розетки из коллекции Гармония» не работает

## Корневая причина — Cap=6 отсекает базовый запрос

Классификатор работает правильно: `category="розетка"`, `modifiers=["Гармония"]`.

Но в коде (строки 3121-3136) запросы формируются так: **сначала все комбинации modifier+category, потом базовые варианты**. С 4 синонимами и 1 модификатором получается:

```text
1. "розетка Гармония"        ← в Cap
2. "Гармония розетка"        ← в Cap
3. "розетки Гармония"        ← в Cap
4. "Гармония розетки"        ← в Cap
5. "электророзетка Гармония" ← в Cap
6. "Гармония электророзетка" ← в Cap
--- Cap=6, отсечка ---
7. "розетка"                 ← НЕ ПОПАДАЕТ
8. "розетки"                 ← НЕ ПОПАДАЕТ
```

API 220volt.kz не находит текст «Гармония» в названиях розеток (коллекция хранится в options, не в title) → все 6 запросов возвращают 0 → Pass 1 пуст → `resolveFiltersWithLLM` не вызывается → система падает на LLM 1 fallback, который тоже не фильтрует по коллекции.

## Решение

Изменить порядок формирования запросов (строки 3121-3136): **базовый запрос без модификаторов должен всегда попадать в Cap**, чтобы Pass 1 гарантированно нашёл товары категории, а `resolveFiltersWithLLM` уже отфильтровал по характеристикам.

Новая логика:
1. Первым идёт базовый запрос категории (`"розетка"`) — он гарантирует наполнение Pass 1
2. Затем комбинации с модификаторами — на случай если API найдёт по тексту
3. Остальные базовые варианты

```text
1. "розетка"                 ← ГАРАНТИРОВАН
2. "розетка Гармония"        ← попытка найти по тексту
3. "Гармония розетка"
4. "розетки Гармония"
5. "розетки"                 ← расширение базы
6. "Гармония розетки"
```

Так Pass 1 найдёт 30+ розеток → `resolveFiltersWithLLM` увидит опцию «Коллекция: Кварта, ГАРМОНИЯ, Октава...» → Pass 2 отфильтрует по `kollekciya=ГАРМОНИЯ`.

## Технические изменения

**Файл**: `supabase/functions/chat-consultant/index.ts`, строки 3121-3136

Переписать порядок `orderedQueries`:
- Сначала добавить `effectiveCategory` (базовый запрос)
- Затем чередовать: combo с modifier, base variant, combo, base...
- Это гарантирует, что при любом Cap хотя бы 1 базовый запрос попадёт в Pass 1

## Что НЕ трогаем
- `resolveFiltersWithLLM` — работает (удлинитель 50м доказал)
- Промпт классификатора — работает (category="розетка", modifiers=["Гармония"] — правильно)
- Cap=6 — не меняем значение, только порядок запросов
- Title-first, price-intent, replacement — не затронуты

## Ожидаемый результат
- Pass 1 находит розетки по базовому запросу
- `resolveFiltersWithLLM` видит схему с коллекциями → `kollekciya=ГАРМОНИЯ`
- Pass 2 возвращает розетки серии Гармония
