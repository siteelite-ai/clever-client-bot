

# Минимальный фикс: новый чат — пустые слоты. Без TTL и кнопок.

## Принцип

Если в UI виджета истории нет (только приветствие или пусто) — слотов быть не должно. Точка. Никаких таймеров, никаких кнопок, никакого UX. Один инвариант на старте.

## Где правка

**`public/embed.js`**, блок восстановления state из `sessionStorage` (строки ~25-39).

После загрузки сохранённого state — одна проверка:

```js
// Инвариант: слоты не могут существовать без пользовательских сообщений.
// Если в истории нет ни одной user-реплики — это «новый чат», слоты сбрасываем.
const hasUserMessages = conversationHistory.some(m => m.role === 'user');
if (!hasUserMessages) {
  dialogSlots = {};
  sessionId = 'session_' + Math.random().toString(36).slice(2, 11) + '_' + Date.now();
}
```

Логика:
- История = только приветствие ассистента → пользователь ничего не писал → это начало → слоты пустые + новый sessionId
- История содержит хотя бы одно сообщение `role=user` → это живое продолжение → слоты сохраняем

`sessionId` тоже пересоздаём, чтобы сервер видел реально новую беседу, а не зомби-id из прошлой вкладочной сессии.

## Что НЕ делаем

- Не вводим TTL
- Не добавляем кнопку «Новый диалог»
- Не трогаем `ChatWidget.tsx`
- Не трогаем сервер
- Не трогаем slot-resolver, classifier, pipeline
