
## Решение проблемы кэширования embed.js

### Корневая причина
Скрипт `embed.js` загружается без cache-busting параметров. Браузер (и CDN) кэшируют файл, и после паблиша клиент продолжает использовать старую версию.

### Решение: версионирование через query-параметр

**Безопасный подход** — добавить в `embed.js` самому механизм автоматического cache-busting, чтобы не заставлять заказчика менять код на своём сайте.

**Файл: `public/embed.js`** — в самом начале скрипта добавить авто-обновление:

```javascript
// В начале IIFE, до всего остального кода:
// Автоматическое обновление скрипта — проверяем, загружен ли с актуальной версией
(function() {
  var script = document.querySelector('script[src*="embed.js"]');
  if (script && !script.src.includes('_v=')) {
    // Перезагружаем скрипт с timestamp для cache-busting
    var newScript = document.createElement('script');
    newScript.src = script.src + (script.src.includes('?') ? '&' : '?') + '_v=' + Math.floor(Date.now() / 300000);
    // 300000ms = 5 минут — кэш обновляется каждые 5 минут
    script.parentNode.removeChild(script);
    document.body.appendChild(newScript);
    return; // Прекращаем выполнение текущего скрипта
  }
  
  // ... весь остальной код виджета ...
})();
```

**Файл: `src/pages/Integration.tsx`** — обновить код для встраивания, добавив cache-busting в рекомендуемый сниппет:

```html
<script src="https://clever-client-bot.lovable.app/embed.js?v=2" async></script>
```

### Как это работает
1. Клиент загружает `embed.js` (возможно из кэша)
2. Скрипт проверяет: есть ли `_v=` в URL? Нет → удаляет себя и перезагружает с timestamp-параметром (`_v=` округлённый до 5 минут)
3. Браузер загружает свежую версию, т.к. URL изменился
4. При повторном выполнении `_v=` уже есть → скрипт работает нормально
5. Каждые 5 минут timestamp меняется → гарантия свежести

### Почему безопасно
- Не меняет логику виджета — только механизм загрузки
- Заказчику не нужно менять код на своём сайте
- 5-минутное окно кэширования — баланс между свежестью и нагрузкой
- Если механизм не сработает — виджет всё равно загрузится (старая версия)

### Файлы для изменения
1. `public/embed.js` — добавить auto-reload логику в начало
2. `src/pages/Integration.tsx` — обновить embed-сниппет с `?v=2`
