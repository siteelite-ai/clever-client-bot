

# План: Показывать 3 товара сразу + предлагать уточнение

## Проблема

Когда найдено >7 товаров, бот задаёт уточняющий вопрос БЕЗ показа товаров. Пользователь ничего не видит и должен вслепую отвечать. Это неудобно и ненужно — у нас уже есть товары, можно сразу показать первые 3.

## Решение

Изменить инструкцию для финального LLM (строка 3844-3852): вместо "задай вопрос ИЛИ покажи" → "ОБЯЗАТЕЛЬНО покажи первые 3, затем предложи уточнить".

## Изменение

**Файл:** `supabase/functions/chat-consultant/index.ts`, строки 3844-3852

Заменить текущий промпт для >7 товаров:

```typescript
productInstructions = `
🎯 НАЙДЕНО ${productCount} ТОВАРОВ ПО НАЗВАНИЮ:
${productContext}

📋 ОБЯЗАТЕЛЬНЫЙ ФОРМАТ ОТВЕТА:
1. Покажи ПЕРВЫЕ 3 наиболее релевантных товара: название, цена, наличие, ссылка
2. Скажи: "Всего нашлось ${productCount} вариантов."
3. Предложи сузить выбор: "Если хотите, могу подобрать точнее — подскажите [тип/характеристика/бренд]"
- Ссылки копируй как есть в формате [Название](URL) — НЕ МЕНЯЙ URL!
- ВАЖНО: если в названии товара есть экранированные скобки \\( и \\) — СОХРАНЯЙ их!
- Тон: профессиональный, без давления
- 🚫 НЕ задавай уточняющий вопрос БЕЗ показа товаров. Всегда сначала показывай 3 товара!`;
```

## Ожидаемый результат

```text
Было:
  Бот: "Нашёл 20 розеток. Какой тип вам нужен: электрическая, компьютерная, телефонная?"
  (пользователь ничего не видит)

Станет:
  Бот: "Вот 3 розетки из коллекции Гармония:
  1. Розетка Гармония белая — 1200₸ [ссылка]
  2. Розетка Гармония бежевая — 1350₸ [ссылка]
  3. Розетка Гармония серая — 1100₸ [ссылка]
  Всего нашлось 20 вариантов. Если хотите, могу подобрать точнее — подскажите тип (электрическая, компьютерная) или способ монтажа."
```

Слот product_search по-прежнему создаётся → при уточнении фильтры накапливаются как раньше.
