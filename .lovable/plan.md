

# Вердикт Системного Архитектора

## Привлечённые роли

- **System Architect** (надроль) — целостность решения, отсутствие «костылей»
- **Knowledge Base Quality Auditor (#6)** — актуальность данных
- **RAG Quality Auditor (#1)** — фильтрация при поиске

---

## Оценка предложенного плана

**Вердикт: ПРИНИМАЮ с уточнениями.**

План добавления `valid_from` / `valid_until` — это **системное решение**, не заплатка. Оно работает на уровне данных, а не на уровне хаков в промпте. Любой контент с истёкшим сроком автоматически исчезает из выдачи RAG — без участия LLM.

### Что правильно в плане
- Nullable поля — бессрочный контент (статьи, нормы) не требует дат
- Фильтрация на уровне SQL RPC — просроченные записи не попадают ни в vector, ни в FTS результаты
- Авто-извлечение дат при импорте — минимизирует ручную работу

### Уточнение архитектора: НЕ НУЖЕН отдельный «бейдж Просрочено» в UI

Просроченные акции не должны висеть в базе как «видимые, но помеченные». Они должны **мягко скрываться** из поиска, но оставаться в таблице для истории. В UI — простой фильтр «Показать просроченные» (toggle), по умолчанию выключен.

## Данные для скрипта бэкфилла

Найдено **8 акций** в базе. Все содержат даты в формате «с DD.MM.YYYY по DD.MM.YYYY» прямо в тексте:

| Акция | Период | Статус |
|---|---|---|
| Акция CHINT | 01.07.2023 — 07.07.2024 | Просрочена |
| Накопительная CHINT | 01.06.2024 — 31.07.2024 | Просрочена |
| Лето с IEK | 01.06.2024 — 31.07.2024 | Просрочена |
| Скидка на все! | 01.05.2021 — 30.06.2023 | Просрочена |
| Доставим до двери! | 01.01.2025 — 28.02.2025 | Просрочена |
| Скидка пенсионерам | 01.01.2025 — 31.12.2030 | **Активна** |
| Navigator SUPERVISION | 04.04.2023 — 30.06.2023 | Просрочена |
| Светильники Meson | 20.09.2024 — 15.10.2024 | Просрочена |

Из 8 акций **только 1 активна** (скидка пенсионерам). Остальные 7 бот может выдать как действующие.

## Итоговый план реализации

### Шаг 1: Миграция БД
Добавить `valid_from` (timestamptz, nullable) и `valid_until` (timestamptz, nullable) в `knowledge_entries`.

### Шаг 2: Бэкфилл скриптом
SQL UPDATE для 8 записей — проставить `valid_from` и `valid_until` из текста контента. Даты известны (таблица выше), можно сделать одним UPDATE.

### Шаг 3: Фильтрация в RPC-функциях
Добавить `WHERE (ke.valid_until IS NULL OR ke.valid_until > now())` в:
- `search_knowledge_hybrid`
- `search_knowledge_fulltext`
- `search_knowledge_chunks_hybrid`

### Шаг 4: Авто-извлечение дат при импорте URL
В `knowledge-process/index.ts` → `scrape_url`: regex-извлечение дат формата `с DD.MM.YYYY по DD.MM.YYYY` из скраченного контента → запись в `valid_from` / `valid_until`.

### Шаг 5: UI — редактирование и фильтрация
- В `KnowledgeBase.tsx`: toggle «Показать просроченные» (по умолчанию OFF), два date-picker поля в форме добавления
- В `EntryViewDialog.tsx`: отображение и редактирование дат действия
