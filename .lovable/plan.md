

# План: Улучшение category-first — извлечение характеристик и фильтров

## Проблемы

### Проблема 1: «черная двухместная розетка» → 1 результат
Микро-LLM определила `category="двухместная розетка"`. Category-first нашел товары через `searchProductsMulti`, но передал кандидатов **без `option_filters`**. Механизм двухпроходной фильтрации (Pass 2 с `discoverOptionKeys`) работает только когда `option_filters` есть в кандидатах. Цвет «черная» потерялся — бот показал что нашел, а основная LLM получила уже `articleShortCircuit=true` и просто описала найденное.

### Проблема 2: «какие есть розетки из коллекции Гармония» → 0 результатов
Классификатор извлек `category="розетка"`, но «Гармония» — это серия/коллекция, которая есть в **названиях товаров** на 220volt.kz («серия ГАРМОНИЯ, РС16-237»). Если искать `"розетка Гармония"`, API найдет 13 товаров. Но `generateCategorySynonyms` получил только «розетка» и сгенерировал синонимы типа «розетки», «электророзетка» — без «Гармония».

### Корневая причина (общая)
Category-first берет только `product_category` из классификатора и теряет остальной контекст запроса (цвет, серия, коллекция, характеристики). Нужно системно извлекать **все атрибуты** из сообщения.

## Решение

### Расширить промпт классификатора — добавить поле `search_modifiers`

Вместо жесткого списка атрибутов (цвет, серия) — одно универсальное поле, куда микро-LLM складывает всё, что не является категорией, но нужно для поиска.

В `classifyProductName()` (строка ~614), добавить задачу 5:

```
Задача 5: search_modifiers — извлеки ВСЕ дополнительные характеристики, 
которые пользователь упоминает помимо категории товара.
Это может быть: цвет, серия/коллекция, бренд, материал, тип монтажа, 
степень защиты, или любое другое уточнение.

Примеры:
- "черная двухместная розетка" → product_category="двухместная розетка", search_modifiers=["черная"]
- "розетки из коллекции Гармония" → product_category="розетка", search_modifiers=["Гармония"]
- "накладная розетка IP44 белая" → product_category="розетка", search_modifiers=["накладная", "IP44", "белая"]
- "кабель для улицы" → product_category="кабель", search_modifiers=["для улицы"]
- "светодиодная лампа E27 теплая" → product_category="лампа", search_modifiers=["светодиодная", "E27", "теплая"]

Ответь JSON: {..., "search_modifiers": ["modifier1", "modifier2"]}
```

### В category-first branch — использовать modifiers двумя способами

**a) Добавить modifiers к синонимам (для попадания через текстовый поиск)**
Каждый синоним категории комбинируется с каждым модификатором:
- Синонимы: `["двухместная розетка", "2-местная розетка", "розетка двойная"]`
- Модификатор: `["черная"]`
- Итого кандидаты: `["двухместная розетка", "черная двухместная розетка", "2-местная розетка", "черная розетка", "розетка двойная"]`

Для «Гармония»: `["розетка Гармония", "розетка Гармония", "2-местная розетка Гармония"]` — API найдет по «Гармония» в названии товара.

**b) Преобразовать modifiers в option_filters для Pass 2**
Передать modifiers как `option_filters` в `searchProductsMulti`, чтобы `discoverOptionKeys` мог сопоставить «черная» → API ключ цвета, «IP44» → ключ защиты.

```
candidates = categoryVariants.map(q => ({
  query: q,
  option_filters: { modifier1: value1, modifier2: value2 }
}))
```

Это задействует уже существующий Pass 2 с `discoverOptionKeys`.

### Что НЕ сломается
- Если `search_modifiers` пустой — category-first работает как сейчас (только синонимы)
- Title-first, price-intent, replacement — не затронуты
- Основной LLM pipeline с его `option_filters` — не затронут
- `discoverOptionKeys` уже умеет fuzzy-match ключей — модификаторы пройдут через него автоматически

### Ожидаемый результат

**«черная двухместная розетка»:**
1. Классификатор → `category="двухместная розетка"`, `search_modifiers=["черная"]`
2. Синонимы: `["двухместная розетка", "2-местная розетка", "розетка двойная"]`
3. Кандидаты с модификатором: `["двухместная розетка", "черная розетка", ...]` + `option_filters={"цвет":"черная"}`
4. Pass 1 находит розетки → Pass 2 фильтрует по цвету → несколько черных розеток

**«розетки из коллекции Гармония»:**
1. Классификатор → `category="розетка"`, `search_modifiers=["Гармония"]`
2. Кандидат: `"розетка Гармония"` → API находит 13 товаров серии ГАРМОНИЯ
3. Все ≤15 → `articleShortCircuit=true` → показать все
