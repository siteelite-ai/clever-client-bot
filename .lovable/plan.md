

# Причина: объём отфильтровывается из характеристик

## Что происходит

API 220volt.kz **возвращает** характеристику «Объем» для кабеля АВВГ 2×2.5. Но в коде есть список `EXCLUDED_OPTION_PREFIXES` (строка 2723), который фильтрует «мусорные» характеристики перед отправкой в LLM. В этом списке:

```
строка 2733: 'obyem',
строка 2734: 'obem',
```

Эти префиксы исключают **все** поля с ключами `obyem*` и `obem*` — то есть именно «Объем, м3» с значением 0.000077. LLM получает характеристики без объёма → правильно говорит «не указан».

Вес (`0.113`) проходит, потому что его ключ (`ves`) не в списке исключений.

## Решение

Удалить `'obyem'` и `'obem'` из массива `EXCLUDED_OPTION_PREFIXES` (строки 2733-2734).

**Файл:** `supabase/functions/chat-consultant/index.ts`

После этого LLM будет видеть характеристику «Объем, м3: 0.000077» в данных товара и сможет применить формулу расчёта.

## Объём изменений
2 строки удалить в одном файле. Передеплой edge function.
