

# План: Улучшить промпт resolveFiltersWithLLM для смыслового резолва

## Проблема

Скриншот показывает: на запрос «черная двухместная розетка» бот нашёл кремовую розетку КВАРТА. Pass 1 работает (розетки находятся), но `resolveFiltersWithLLM` не справляется с маппингом модификаторов на характеристики. Текущий промпт (строки 2140-2157) слишком сухой — он просто говорит «соотноси по смыслу», но не объясняет LLM **процесс рассуждения**.

## Что хочет пользователь

Промпт должен описать **алгоритм мышления** LLM, а не давать примеры:

1. Посмотри на модификаторы пользователя — пока просто запомни их как «намерения», не привязывая к конкретным характеристикам
2. Пройдись по ВСЕМ характеристикам схемы — для каждой посмотри её название (caption) и все доступные значения
3. Для каждого модификатора определи: какая характеристика по смыслу соответствует этому намерению? Учитывай физический смысл, единицы измерения, контекст категории
4. Найдя подходящую характеристику — посмотри, как именно записаны её значения (цифрами, словами, сокращениями) и выбери точное значение из списка
5. Если модификатор не подходит ни к одной характеристике — пропусти его

## Конкретные изменения

**Файл**: `supabase/functions/chat-consultant/index.ts`, строки 2140-2157

Заменить системный промпт в `resolveFiltersWithLLM` на новый, описывающий пошаговый процесс рассуждения:

```
Ты резолвер фильтров товаров интернет-магазина электротоваров.

ЗАДАЧА: Сопоставь модификаторы из запроса пользователя с реальными характеристиками товаров и подбери точные значения фильтров.

СХЕМА ХАРАКТЕРИСТИК КАТЕГОРИИ:
{schemaText}

МОДИФИКАТОРЫ ПОЛЬЗОВАТЕЛЯ:
{modifiers}

АЛГОРИТМ ДЕЙСТВИЙ:
1. Прочитай модификаторы пользователя. Каждый модификатор — это намерение: 
   пользователь хочет товар с определённым свойством, но не знает, как именно 
   это свойство называется в каталоге.
2. Пройдись по каждой характеристике в схеме. Для каждой смотри на её 
   название (в скобках) и на все доступные значения. Определи, что эта 
   характеристика описывает физически.
3. Для каждого модификатора найди характеристику, которая описывает то же 
   самое свойство. Учитывай: единицы измерения, физический смысл в контексте 
   данной категории товаров, возможные синонимы и сокращения.
4. Найдя подходящую характеристику, посмотри на формат её значений — 
   они могут быть записаны цифрами, словами, с единицами измерения, 
   сокращениями. Выбери из списка то значение, которое точно соответствует 
   намерению пользователя. Возвращай значение В ТОЧНОСТИ как в схеме.
5. Если модификатор не соответствует ни одной характеристике — не включай 
   его в результат. Не угадывай.

Ответь СТРОГО в JSON: {"filters": {"api_key": "exact_value", ...}}
Если ни один модификатор не удалось сопоставить — верни {"filters": {}}
```

Добавить инструкцию по контексту диалога, чтобы новый запрос с категорией не тянул модификаторы из предыдущих сообщений:

```
КОНТЕКСТ ДИАЛОГА: Если текущее сообщение — самостоятельный новый запрос 
(содержит категорию товара или название), извлекай ВСЕ поля ТОЛЬКО из 
текущего сообщения. НЕ переноси category, modifiers, product_name из 
предыдущих сообщений. Используй историю ТОЛЬКО для коротких ответов-уточнений 
(1-3 слова: «давай», «телефонную», «да»).
```
