

# План: Фильтрация «thinking»-сообщений из истории API

## Проблема

Сейчас в `ChatWidget.tsx` при отправке сообщения в API включаются все сообщения из состояния, в том числе служебные: фразы ожидания («Сейчас подберу варианты»), плейсхолдеры анимации (`__TYPING__`). Микро-LLM получает загрязнённую историю и неправильно классифицирует интент пользователя.

В `embed.js` этой проблемы нет — там thinking-фразы не попадают в `conversationHistory`.

## Что нужно сделать

### Файл: `src/components/widget/ChatWidget.tsx`

В месте, где формируется массив `apiMessages` для отправки на сервер, добавить фильтр — исключить:

- Сообщения с контентом `__TYPING__`
- Сообщения с ID, начинающимся на `thinking-`, `typing-`, `typing2-`

Это одно изменение в ~5 строк. Визуально ничего не меняется — пользователь по-прежнему видит фразу ожидания и анимацию. Но в API уходит только реальный диалог, и классификатор работает корректно.

## Объём

Один файл, одно изменение. Деплой edge function не нужен.
