

# План: Исправить `extractModifiersFromProduct`

## Проблема
Строка 735: `.replace(/\s+/g, '')` склеивает многословные значения. LLM не может распарсить `"холодныйбелый"` и возвращает обрезанный JSON.

## Исправление (файл: `supabase/functions/chat-consultant/index.ts`, строки 707-741)

### 1. Убрать склейку пробелов для текстовых значений
Оставить пробелы как есть. Компактификация (удаление пробела) — только для чисел с единицами: `"100 Вт"` → `"100Вт"`.

### 2. Добавить единицы измерения к голым числам
Если значение — число, а caption содержит подсказку:
- caption содержит `мощность`/`Вт` и value=`100` → `"100Вт"`
- caption содержит `защит`/`IP` и value=`67` → `"IP67"`
- caption содержит `напряжение`/`В` и value=`220` → `"220В"`

### 3. Фильтровать мусор
Пропускать значения: `"нет"`, `"да"`, `"не требуется"`, а также бесполезные числа без контекста (длина `55` — не критична для подбора аналога).

### 4. Ограничить количество
Максимум 8 модификаторов, чтобы не перегружать LLM и не получать обрезанный JSON.

## Ожидаемый результат

Вместо:
```
[серый, холодныйбелый, 55, нет, алюминиевыйсплав, поликарбонат, 100, 220, 67, нет, LED, дляосвещенияулиц]
```

Будет:
```
[серый, холодный белый, 100Вт, 220В, IP67, LED, для освещения улиц]
```

LLM сопоставит эти значения со схемой → resolved filters заполнятся → STAGE 2 вернёт аналоги с нужными характеристиками.
