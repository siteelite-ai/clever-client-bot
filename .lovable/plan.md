

# План: Выбор категории по совместимости модификаторов

## Суть идеи

Сейчас: поиск `category="Автоматы"` → API возвращает только "Автоматы защиты двигателя" → модификаторы не матчат → fallback.

Как должно быть: поиск `query="автомат"` (без жёсткой привязки к category) → получаем товары из РАЗНЫХ категорий ("Автоматы защиты двигателя", "Автоматические выключатели", и т.д.) → перебираем бакеты → выбираем ту категорию, чья схема лучше всего матчит модификаторы ("1 полюсные", "16 ампер").

## Изменения

### Файл: `supabase/functions/chat-consultant/index.ts`

**Строки ~3284-3331** — изменить стратегию первичного поиска:

1. **Два параллельных поиска**: и по `category=pluralCategory`, и по `query=effectiveCategory` — объединить результаты для максимального покрытия категорий
2. **Бакетизация по категориям** — уже есть (строки 3308-3314)
3. **Перебор бакетов с resolveFiltersWithLLM** — для каждого бакета проверяем, сколько модификаторов резолвится. Выбираем бакет с максимальным количеством resolved
4. **Убрать текущий fallback** (строки 3335-3406) — он больше не нужен, т.к. перебор бакетов теперь основная логика

```text
Поток:
  query="автомат" (без category) → 50 товаров
     ├── Бакет "Автоматы защиты двигателя" (30 шт)
     │   └── resolveFilters → 0 resolved ✗
     ├── Бакет "Автоматические выключатели" (15 шт)
     │   └── resolveFilters → 2 resolved (полюс=1, ток=16) ✓ ← WINNER
     └── Бакет "Другое" (5 шт)
         └── skip (не проверяем, уже нашли)

  STAGE 2: category="Автоматические выключатели" + filters → точные результаты
```

**Тот же паттерн** применить в replacement-блоке (~строка 3504).

### Деплой
Edge function `chat-consultant`.

## Ожидаемый результат

Запрос "подбери автоматы 1 полюсные на 16 ампер":
1. `query="автомат"` → товары из разных категорий
2. Перебор бакетов → "Автоматические выключатели" побеждает (2 resolved из 2)
3. STAGE 2 с `category="Автоматические выключатели"` + `kolichestvo_polyusov=1, nominalynyy_tok=16`
4. Точные результаты

~50 строк переработки логики category-first + деплой.
