

## Доработка отображения товаров: цена 0 и «Под заказ»

### Проблема (видно на скриншоте)
1. Товар с ценой **0 ₸** отображается как `*0* ₸` — бессмысленно для клиента
2. Товары без наличия показывают **«В наличии: Под заказ»** — противоречивая формулировка. Нужно просто **«Под заказ»**

### Изменения

**Файл:** `supabase/functions/chat-consultant/index.ts`

**Изменение 1: Фильтрация товаров с ценой 0 (строка ~1209)**

После `uniqueProducts` — отфильтровать товары с `price === 0`:
```typescript
const filteredProducts = uniqueProducts.filter(p => p.price > 0);
```
Дальнейшая сортировка и лимит — по `filteredProducts`. Если после фильтра пусто — вернуть оригинальный список (на случай если ВСЕ товары с ценой 0).

**Изменение 2: Упростить текст наличия (строки 1598-1607)**

Текущая логика:
- Есть склады → `В наличии: Да (X шт.)` + города
- Нет складов → `В наличии: Да/Под заказ`

Новая логика:
- Есть склады с остатком → `Остатки по городам: Алматы: 100 шт., Актобе: 81 шт.` (без «В наличии: Да»)
- Нет остатков (amount=0, все склады пусты) → просто `Под заказ` (без префикса «В наличии:»)

**Деплой:** Edge function `chat-consultant`
