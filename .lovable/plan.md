

# План: Исправление контента о доставке в базе знаний

## Корневая причина

Запись «Доставка | 220 volt» (id: `88758557-28d2-4fd3-980d-8a81b2b5008b`) содержит двусмысленный текст:

```
СДЭК и Казпочта. Бесплатно, из пунктов самовывоза;
```

LLM интерпретирует это как «СДЭК/Казпочта = бесплатно», хотя реальный смысл — три отдельных способа доставки.

## Решение

Обновить `content` этой записи в базе знаний, переписав блок о способах доставки в чёткую структуру:

```
Способы доставки:

1. **Самовывоз из фирменных магазинов 220volt** — БЕСПЛАТНО.
   Забрать заказ можно из любого фирменного магазина 220volt.

2. **СДЭК (пункты выдачи и курьерская доставка)** — ПЛАТНО.
   Стоимость и сроки рассчитываются по тарифам СДЭК.
   Срок хранения заказа на пункте выдачи — 7 календарных дней.

3. **Казпочта** — ПЛАТНО.
   Стоимость и сроки рассчитываются по тарифам Казпочты.

4. **Индивидуальная доставка (от 50 кг или по договору)** —
   сроки и стоимость определяются при подтверждении заказа.

5. **Не подошёл ни один вариант** — свяжитесь с нами для обсуждения
   индивидуальных условий.
```

## Что меняется

| Действие | Деталь |
|---|---|
| SQL UPDATE | Обновить `content` записи `88758557-...` — переписать блок о способах доставки в однозначную структуру |
| Пересчёт эмбеддинга | После обновления текста вызвать `knowledge-process` для перегенерации эмбеддинга этой записи |

## Риски

| Риск | Предотвращение |
|---|---|
| Потеря другой информации из записи | Меняем только блок «Способы доставки», остальной текст сохраняется |
| Эмбеддинг не обновится | Явно вызываем пересчёт после UPDATE |

Один SQL-запрос + один вызов Edge Function. Изменений в коде нет.
