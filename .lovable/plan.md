

# План: Мгновенная «фраза ожидания» перед поиском

## Идея

Сейчас пользователь отправляет сообщение и видит только анимацию трёх точек 5-10 секунд, пока пайплайн работает. Вместо этого — сразу показать контекстную фразу типа «Сейчас посмотрю, что есть по розеткам...» или «Минутку, проверю информацию...», а под ней — анимация загрузки. Когда реальный ответ начнёт стримиться, фраза заменяется на него.

## Как это работает

### Серверная часть (chat-consultant/index.ts)

Сразу после получения запроса, ДО запуска пайплайна, отправляем первый SSE-чанк с событием `thinking`. Для этого нужно перейти на ручной стриминг — создать `ReadableStream` в самом начале обработки запроса и вернуть `Response` сразу, а пайплайн запустить внутри `start()` / `pull()`.

Фразы зависят от intent (определяется быстро по ключевым словам, ещё до микро-LLM):

- Товарные слова в сообщении → «Сейчас подберу варианты...», «Ищу в каталоге...», «Секунду, смотрю наличие...»
- Нет товарных слов → «Сейчас проверю информацию...», «Минутку...», «Уточняю детали...»

SSE-формат:
```
data: {"thinking":"Сейчас подберу варианты..."}
```

### Клиентская часть (ChatWidget.tsx + embed.js)

1. При получении события `thinking` — показать сообщение-бот с текстом фразы + анимация загрузки под ним
2. Когда приходит первый реальный `delta` — заменить фразу на стримящийся ответ

## Объём изменений

- `supabase/functions/chat-consultant/index.ts` — переход на ручной `ReadableStream` с ранним ответом; массив фраз; отправка `thinking`-события до пайплайна (~30-40 строк)
- `src/components/widget/ChatWidget.tsx` — обработка `thinking`-события в `streamChat`; показ фразы в UI (~15 строк)
- `public/embed.js` — аналогичная обработка `thinking`-события (~10 строк)
- 1 деплой edge function

## Важные детали

- Фразы рандомные из пула (3-5 вариантов на категорию), чтобы не повторялись
- Фраза исчезает при первом реальном токене — не остаётся в истории сообщений
- Если ответ приходит быстро (<1 сек), фраза может даже не успеть отобразиться — это нормально
- Никакой логики на ключевых словах для роутинга — только для выбора тона фразы ожидания
